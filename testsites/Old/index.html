The purpose of this website is to the demonstrate the different CSS elements that are commonly used in web design. The important thing to always keep in mind is sticking to a theme; a theme is usually established in the design phase of making a webpage. A design phase includes sitemaps, wireframes, and moodboards. I will delve further into each part of the design phase on this page.

Designing a Webpage

Sitemap

A sitemap is a visual representation of how all your pages link together and where each page can take you to. This will include every link that is responsible for opening a new page. Sitemap’s are important as it gives you some form of direction when making your site and ensures all pages are easily accessed when it comes to the final project.

Moodboards

Moodboards are a design technique for all the visual elements of a webpage. This includes pictures, a colour palette, buttons, fonts, text styles, logos etc. This helps your webpage keep a unified look, making each page similar so the site has a theme that is recognisable to the user. Without a theme colours can clash, and one page could look like it’s from a different site entirely.

Wireframes

Wireframes detail what each page will look like; which involves determining the box model of the page, details of sizes and styles. This is just a simple outline design as all the details for the styles should be in the wireframes. Style details can include colours, font sizes, dimensions and many more. The way to tell if the wireframe is to a satisfactory standard is to think about if it was given to another person to make the website from the wireframes; if they can make the website exactly as designed then the wireframe worked.
